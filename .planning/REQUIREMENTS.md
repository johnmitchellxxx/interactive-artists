# Requirements: Interactive Artists Team Website

**Defined:** 2026-03-11
**Core Value:** Potential clients should immediately understand who we are, be impressed by our work, and know how to hire us.

## v1 Requirements

### Gallery

- [ ] **GALL-01**: Visitor can view project images in a carousel or grouped image layout on the homepage
- [ ] **GALL-02**: Project images animate (e.g. scale, reveal, or overlay) when the visitor mouses over them

### Team

- [ ] **TEAM-01**: Visitor can view each artist's photo, name, and bio on the site

### Trust

- [ ] **TRST-01**: Visitor can view a grid or row of past client/brand logos
- [ ] **TRST-02**: Visitor can view press mentions or media coverage (quotes, publication names, or links)

### Contact

- [ ] **CONT-01**: Visitor can submit a contact form with their name and email (required fields)
- [ ] **CONT-02**: Visitor can optionally select a project type from: Installation / Public art / Corporate event
- [ ] **CONT-03**: Visitor can optionally describe their project or idea in a text area
- [ ] **CONT-04**: Team receives the form submission via email

## v2 Requirements

### Portfolio (expanded)

- **PORT-01**: Each project has its own detail page with photos, video, and case study text
- **PORT-02**: Visitor can filter projects by category (Installations / Public & Outdoor / Corporate Events)
- **PORT-03**: Video embeds (Vimeo/YouTube) on project pages

### Team (expanded)

- **TEAM-02**: A collective About page with team overview, mission, or story

### Contact (expanded)

- **CONT-05**: Optional budget range dropdown on contact form to help qualify leads

## Out of Scope

| Feature | Reason |
|---------|--------|
| Blog / news feed | Not a priority; tends to go stale |
| Online booking / scheduling | Contact form is sufficient for bespoke work |
| E-commerce / print sales | Not the business model |
| Self-hosted video | Bandwidth cost and performance risk; use embeds |
| Multilingual support | Not mentioned; adds significant scope |
